## 🌡️ 温度是什么？

### 一句话解释

**温度 = 记忆与当前任务的"相关程度"**

## 🔧 如何计算温度？

### 方法对比

| 方法 | 原理 | 性能 | 适用 |
|------|------|------|------|
| **向量相似度** ❌ | 文字转数字向量，计算夹角 | ⚠️ 慢（需要 GPU） | 大模型/有钱 |
| **关键词匹配** ✅ | 看有多少相同的词 | ✅ 快（字符串比较） | 廉价服务器 ✅ |

**本项目选择**: 关键词匹配（便宜、快速、够用）

---

### 计算过程（两步走）

#### 步骤 1: 路径匹配（权重 60%）

```
记忆路径：/person/九斤/preferences
当前上下文："约九斤去花卉小镇拍照"

提取路径关键词：["person", "九斤", "preferences"]
匹配结果：
- "person" 不在上下文中 → 0 分
- "九斤" 在上下文中 → 1 分
- "preferences" 不在上下文中 → 0 分

路径匹配得分 = 1/3 = 0.33
```

#### 步骤 2: 关键词匹配（权重 40%）

```
AI 关键词：["九斤", "乙女游戏", "柏源"]
当前上下文："约九斤去花卉小镇拍照"

匹配结果：
- "九斤" 在上下文中 → 1 分
- "乙女游戏" 不在 → 0 分
- "柏源" 不在 → 0 分

关键词匹配得分 = 1/3 = 0.33
```

#### 综合评分

```
温度 T = 路径得分 × 0.6 + 关键词得分 × 0.4
      = 0.33 × 0.6 + 0.33 × 0.4
      = 0.33
```

## 💡 温度的物理意义

### 热力学类比

```
热力学：热量从高温流向低温
   ↓
记忆系统：记忆从记忆库流向工作记忆

条件：温差 ΔT > 0
   ↓
条件：关联度 T > 阈值（如 0.5）
```

### 实际效果

```python
# 高温度（T > 0.7）
记忆："九斤的联系方式"
上下文："约九斤吃饭"
→ 温度高，容易被提取

# 低温度（T < 0.3）
记忆："MFS 项目计划"
上下文："约九斤吃饭"
→ 温度低，不会被提取
```

## 🎯 温度在四系统中的作用

### 自由能公式

```
G = U - TS

G: 自由能（记忆有效性）
U: 内能（被访问次数，0-100）
T: 温度（关联度，0-1）
S: 熵（争议性，0-1）
```

### 温度的影响

**场景 1: 高内能 + 低温度**
```
记忆："地球是圆的"
U = 10000（被提及无数次）
T = 0.1（与当前任务无关）
S = 0.1（无争议）

G = 10000 - 0.1 × 0.1 × 100 = 9999

结果：虽然自由能高，但因为温度低，不会被提取
```

**场景 2: 低内能 + 高温度**
```
记忆："三月份花卉小镇有花"
U = 5（只提过几次）
T = 0.9（与当前任务高度相关）
S = 0.1（无争议）

G = 5 - 0.9 × 0.1 × 100 = 5 - 9 = -4

结果：虽然内能低，但温度高，
     自由能略负，接近可提取状态
```

## 📊 温度评分标准

| 温度范围 | 等级 | 说明 | 示例 |
|---------|------|------|------|
| **0.7 - 1.0** | 🔥 高相关 | 路径/关键词高度匹配 | "九斤" 在路径和上下文中都出现 |
| **0.4 - 0.7** | 🌤️ 中相关 | 部分匹配 | 只有路径或关键词匹配 |
| **0.1 - 0.4** | ❄️ 低相关 | 少量匹配 | 只有 1 个词匹配 |
| **0.0 - 0.1** | 🧊 无关 | 几乎不匹配 | 完全无关的记忆 |

## 🚀 性能优势

### 轻量级实现

**不用向量库的好处**:
- ✅ 无需 GPU
- ✅ 无需 embedding 模型
- ✅ 计算速度快（毫秒级）
- ✅ 内存占用低
- ✅ 适合廉价云服务器

**性能对比**:

| 方法 | 单次计算耗时 | 内存占用 | 是否需要 GPU |
|------|------------|---------|-------------|
| 向量相似度 | ~50-100ms | ~500MB | ⚠️ 推荐 |
| **关键词匹配** | **~1-5ms** | **~10MB** | **✅ 不需要** |

## 📝 代码示例

### 计算温度

```python
from mfs.free_energy_manager import FreeEnergyManager

fe_mgr = FreeEnergyManager(db_path="mfs.db")

# 计算记忆与上下文的关联度（温度）
context = "约九斤去花卉小镇拍照"
temp_score = fe_mgr._calculate_relevance("memory_001", context)

print(f"温度（关联度）: {temp_score}")
# 输出：温度（关联度）: 0.8
```

### 获取可提取记忆

```python
# 获取所有与当前上下文相关的记忆（自动计算温度）

    context=context,
    limit=10
)

for mem in extractable:
    print(f"{mem['memory_path']}: T={mem['temp_score']:.2f}, G={mem['free_energy_score']:.2f}")
```

## 🎊 总结

**温度系统核心**:
1. **是什么**: 记忆与当前任务的相关程度
2. **怎么算**: 路径匹配 + 关键词匹配（轻量级）
3. **为什么**: 决定记忆是否容易被提取
4. **性能**: 毫秒级，无需向量库

**公式**:
```
温度 T = 路径匹配 × 0.6 + 关键词匹配 × 0.4

取值范围：0.0 - 1.0
```

**效果**:
- 🔥 高温度（T > 0.7）: 容易想起
- 🌤️ 中温度（0.4-0.7）: 可能想起
- ❄️ 低温度（T < 0.4）: 难以想起
